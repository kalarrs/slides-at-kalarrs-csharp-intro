General Confusion

Not everything is "Serverless"

However everyone will refer to it as "Serverless"

Let's Clarify

FaaS

Function as a service (FaaS) is a category of cloud computing services that provides a platform allowing customers to develop, run, and manage application functionalities without the complexity of building and maintaining the infrastructure.

AWS Lambda
Microsoft Functions
Google Cloud Functions

Serverless

Serverless computing is a cloud-computing execution model in which the cloud provider acts as the server, dynamically managing the allocation of machine resources.
Typically by utilizing a FaaS service provider.

#1 Creating functions in
AWS Console!

Create a function in under a minute! Just go to AWS Lambda Console

https://console.aws.amazon.com/lambda/home?region=us-east-1#/home

Pitfalls

  • Runtime languages are easy to get started with, but not compiled languages.
  • C# code can't be edited in the browser. This is because it is a jitted language.
  • Really no instructions on what everything means. You would have to read a lot of docs.
  • Not clear how to upload your own code.

#2 Creating functions in
Visual Studio

Use VS to easily create and upload functions by using the AWS Toolkit for Visual Studio

https://aws.amazon.com/visualstudio/

prerequisites

Update Visual Studio to latest version

Update .NET SDK to latest version

AWS Console
VS
Visual Studio Plugin

Note: You will need to setup AWS credentials to use the plugin.

2a - From AWS Template

Simply create a new project from visual studio and choose the AWS template. It's just that easy.

Pitfalls

  • Simple functions are easy to deploy, however, unable to tell what code is deployed.
  • Difficult to coordinate with teams.
  • No real security, everyone is all powerful.
  • Difficult to make complex event triggers.
  • Limited local debugging.

#2-b From AWS serverless template

Simply create a new project from visual studio and choose the AWS "serverless" template. It's just that easy (again).

Works by using Cloud​Formation

Pitfalls

  • You have to know cloud formation in order to expand the number of endpoints.
  • Limited local debugging.
